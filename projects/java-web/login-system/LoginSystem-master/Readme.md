## 💻 Types of Applications

### 1. Standalone Application

* Runs on a **local machine**
* CLI (Command Line Interface) based
* Example: Console-based Java programs

### 2. Web Application

* Runs on a **browser**
* Based on web technologies (HTML, CSS, JS)
* Source code is deployed on a **Web Server**
* Output is shown in the browser

#### Key Components:

* **Web Server**: Apache Tomcat
* **Servlet**: Java file running on server to handle logic
* **JSP (Java Server Pages)**: Used to generate UI (response)

#### Notes:

* Servlet = Core Java file on server
* JSP = Web page returned to client
* **Struts Framework**: Provides architecture, folder structure, and libraries

---

### 3. Enterprise Application (Large Scale)

* Used for large systems
* Multiple servers are configured

#### Servers Used:

* Web Server (Jboss / GlassFish)
* Database Server
* File Server (FTP)
* Backup Server

## 🌐 What Happens When You Open a Website

Example: `www.greatlearning.com/signin`

1. Request is sent from browser (HTML page)
2. Request goes to server (Tomcat)
3. Server identifies:

   * Static content → HTML/CSS
   * Dynamic content → Servlet
4. Servlet processes the request
5. Response (HTML) is generated
6. Browser renders the UI

---
